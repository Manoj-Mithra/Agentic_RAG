# Local Agentic RAG

A modular, local-first Agentic RAG MVP built with Streamlit, Ollama Qwen3:8B,
Sentence Transformers embeddings, ChromaDB, hybrid retrieval, reranking, and
strict grounded answering.

## Grounding Rule

The assistant must not answer from pretrained knowledge. It may use the LLM for
query understanding, intent detection, query rewriting, planning, summarizing
retrieved context, and formatting. Final answers are based only on retrieved,
crawled, uploaded, or indexed local context.

If no relevant context is available, the app responds:

```text
I do not have enough retrieved information to answer that.
```

Retrieved content is treated as untrusted data. Instructions inside documents
or webpages must be ignored.

## Setup

Install Python dependencies:

```powershell
cd "C:\Users\might\OneDrive\Documents\Agentic RAG\agentic_rag"
python -m pip install -r requirements.txt
```

Install and prepare Ollama:

```powershell
ollama pull qwen3:8b
```

Run the app:

```powershell
streamlit run app.py
```

The first embedding or reranking call downloads:

- `BAAI/bge-small-en-v1.5`
- `BAAI/bge-reranker-base`

## Workflow

1. Upload a PDF, crawl a URL, or paste local text.
2. The app chunks, embeds, and stores content in ChromaDB.
3. Ask a question in chat.
4. The agent rewrites the query for retrieval.
5. It searches Chroma semantically and can add BM25 keyword retrieval.
6. Results are reranked with `BAAI/bge-reranker-base`.
7. The answer is generated by Qwen3:8B using only selected context.
8. Citations and self-check state are shown in the right panel.

## Web Search Policy

Web search is off by default. DuckDuckGo search runs only when the query asks
for online lookup and the UI toggle `Allow web search for this query` is on.

Local vector storage, uploaded documents, pasted text, and user-provided URLs
are preferred before online search.

```text
  core/                Schemas, LLM, embeddings, prompts

```
